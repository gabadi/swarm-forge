# docs/ideas/

These files are **temporary implementation specs**. Each one describes a proposed divergence from upstream that has not yet been implemented.

## Lifecycle

1. **Proposed** — idea file exists, open questions may remain
2. **Ready** — all open questions resolved; file is implementation-ready
3. **Implemented** — delete the idea file; add a condensed entry to the "Current divergences" section of [`docs/adr/0001-fork-divergence.md`](../adr/0001-fork-divergence.md) with: files changed, classification (permanent/temporary), and rebase instruction
4. **Rejected** — delete the idea file; add a one-line struck entry to the "Rejected" list in ADR-0001

Do not keep idea files after implementation. The ADR is the permanent record.

## Status of open questions

See the "Proposed divergences" table in ADR-0001 — the "Open questions" column tracks what remains unresolved per idea.
